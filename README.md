# Online Food Delivery System

An online food delivery system helps customers browse restaurants, place food
orders, make payments, and track deliveries from a single platform.

## Core Users

- **Customer** – searches restaurants, adds meals to cart, places orders, and tracks delivery.
- **Restaurant** – manages menus, accepts orders, and updates preparation status.
- **Delivery Agent** – receives delivery assignments and updates delivery progress.
- **Administrator** – monitors users, restaurants, orders, and platform activity.

## Main Features

### Customer Features

- User registration and login
- Restaurant and menu browsing
- Search and filter for meals
- Add to cart and update quantities
- Place orders with delivery address
- Secure online payment or cash on delivery
- Real-time order status tracking
- Order history and reorder support

### Restaurant Features

- Restaurant profile management
- Menu item creation, update, and removal
- Order acceptance and rejection
- Preparation status updates
- Sales and order monitoring

### Delivery Features

- Delivery partner login
- Assigned order list
- Pickup confirmation
- Delivery status updates
- Delivery history

### Admin Features

- User and restaurant management
- Delivery partner management
- Order monitoring
- Reports and analytics
- Complaint and support handling

## Typical Workflow

1. A customer signs in and selects a restaurant.
2. The customer adds food items to the cart and confirms the order.
3. The restaurant receives and prepares the order.
4. A delivery agent is assigned to pick up the order.
5. The customer tracks the order until it is delivered.

## Functional Requirements

- The system must allow customers to create and manage accounts.
- The system must display restaurants and their menus.
- The system must support cart and checkout functionality.
- The system must record orders and update their statuses.
- The system must support restaurant and delivery management.
- The system must provide administrative control over platform operations.

## Non-Functional Requirements

- Easy-to-use interface
- Fast response time
- Secure authentication and payment handling
- Reliable order tracking
- Scalable design for multiple restaurants and users
